# v76 動作不良原因調査

## 確認したこと

- v69は利用可能だったため、v69を安定基準として確認
- v70〜v75で追加されたポップアップ・追加ボタン処理を確認
- `index.html` / `assets/css/app.css` / `assets/js/app.js` / `gas/Code.gs` を再構成
- `node --check assets/js/app.js` でJavaScript構文確認

## 動かなかった主原因

1. v70以降で、同じ画面描画処理を複数回上書きしていた
2. 追加ボタンと編集ポップアップのクリック処理が複数残っていた
3. PC用・スマホ用で表示と操作処理が分かれ、片方だけ反応しない状態になっていた
4. 画面表示は出ても、イベント処理が最後の上書きで消える構造だった
5. 年度管理・週表示・担当表示・追加編集を後付けしていたため、関数同士が競合していた

## v76での対応

- v70〜v75の後付け処理は使わない
- 追加・編集ポップアップを1つに統一
- 日表示、週表示、担当一覧、変更登録で共通の編集処理を使用
- PC・スマホ共通でイベント委譲によるクリック処理に統一
- GASも年度JSON用に整理
- 古いFEATUREファイルは同梱しない
